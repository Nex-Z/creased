很多团队第一次接触设计 Token 时，会把它理解成变量：颜色变量、字号变量、间距变量。这没有错，但还不完整。

Token 更重要的意义，是把设计决策变成可以被讨论、复用和迁移的语言。`--c-primary` 不只是一个十六进制颜色，它代表品牌主色在整个界面里的使用边界。

当 Token 命名清晰时，组件实现会更稳定。开发者不需要每次都重新挑颜色，设计师也能更容易判断某个视觉变化是否破坏了系统。

好的 Token 系统不是追求数量多，而是让选择更少、更明确。
