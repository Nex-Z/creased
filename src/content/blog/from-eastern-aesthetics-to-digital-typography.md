在中文互联网上，我们常常看到信息密度极高的页面：广告、弹窗、密密麻麻的文字。然而，阅读体验的本质并不在于填满每一寸屏幕，而在于为读者的目光提供一条清晰、舒适的路径。

## 为什么关注排版

好的排版是隐形的。当读者沉浸在内容中，不会注意到字体大小、行间距或段落间距的存在，这恰恰说明设计做对了。相反，糟糕的排版会让人时刻意识到「阅读」本身，产生疲劳感。

> 排版不是装饰，而是沟通的基础设施。好的排版让思想自由流动，坏的排版则在思想和读者之间筑起高墙。

对于中文排版而言，我们面临一些独特的挑战：汉字是方块字，没有大小写之分；中文没有单词间的空格；标点符号的处理规则也与西文不同。这些差异意味着，我们不能简单地将西文的排版规则套用在中文上。

## 字体的选择

字体是排版的灵魂。在中文博客设计中，我通常选择两种字体搭配：

- 标题使用宋体或衬线体，传达书卷气和沉稳感。
- 正文使用清晰的系统无衬线字体，保证屏幕上的可读性。

这种搭配既保留了东方审美中的文化质感，又兼顾了数字屏幕上的阅读效率。

## 间距与节奏

中文正文通常需要更宽松的行间距。我的经验值是 `1.7` 到 `1.9`，配合足够的段间距，让读者不必在密集文本里费力寻找下一行。

段落之间的间距同样重要。现代中文网页更倾向于用空间区分段落，而不是强行套用纸质书中的首行缩进。

## 留白的艺术

留白不是空洞，而是设计的呼吸。在东方美学中，「虚」与「实」的对比是一种核心理念。就像水墨画中，未着墨的部分同样构成画面的一部分。

![简洁的工作空间，少即是多](/images/desk-illustration.png)

在博客排版中，这种理念体现为：内容区域不要太宽，模块之间留出足够的喘息空间，图片与文字之间保持稳定间距，页面四周的 padding 要充裕。

## 代码与文字共存

作为技术博客，代码展示是不可或缺的一部分。代码块应该在视觉上与正文区分开来，但又不能突兀。

```css
:root {
  --font-serif: 'Songti SC', serif;
  --font-sans: system-ui, sans-serif;
  --content-width: 720px;
  --line-height: 1.8;
}

.article-content {
  max-width: var(--content-width);
  margin: 0 auto;
  line-height: var(--line-height);
}
```

行内代码则使用更柔和的处理方式，让它融入正文的阅读节奏中。

## 写在最后

好的博客排版不需要花哨的特效或复杂的布局。它需要的是对细节的关注：字体的选择、间距的调整、色彩的克制、留白的运用。当这些元素和谐配合，读者就能专注于最重要的东西：文字和思想。
